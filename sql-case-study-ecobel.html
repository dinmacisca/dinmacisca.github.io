Query 01
Product Sales Performance Ranking by Revenue & Growth
Ranks all ECO-MAX SKUs by total revenue and calculates month-over-month growth rate to identify top performers and underperforming products requiring strategic review.
product_sales_ranking.sql
MySQL
-- ECO-MAX product sales ranking with growth rate analysis -- Ecobel International | Business Analysis 2024 WITH MonthlySales AS ( SELECT product_name, product_category, DATE_FORMAT(sale_date, '%Y-%m') AS sale_month, SUM(revenue) AS monthly_revenue, SUM(units_sold) AS monthly_units FROM sales_transactions WHERE sale_date BETWEEN '2024-01-01' AND '2024-09-30' GROUP BY product_name, product_category, sale_month ), SalesWithLag AS ( SELECT *, LAG(monthly_revenue) OVER ( PARTITION BY product_name ORDER BY sale_month ) AS prev_month_revenue FROM MonthlySales ) SELECT product_name, product_category, SUM(monthly_revenue) AS total_revenue, SUM(monthly_units) AS total_units, ROUND(AVG( CASE WHEN prev_month_revenue > 0 THEN ((monthly_revenue - prev_month_revenue) / prev_month_revenue) * 100 END ), 1) AS avg_monthly_growth_pct, RANK() OVER ( ORDER BY SUM(monthly_revenue) DESC ) AS revenue_rank FROM SalesWithLag GROUP BY product_name, product_category ORDER BY total_revenue DESC;

Query 02
Customer Segment Analysis — Fleet vs Retail vs Workshop
Segments customers by type to identify the highest-value segments, average order value, and purchase frequency — key insight that identified fleet operators as the most valuable untapped segment.
customer_segment_analysis.sql
BigQuery
-- Customer segmentation analysis to identify high-value target groups -- Supports B2B fleet operator outreach strategy recommendation SELECT c.customer_type, c.territory, COUNT(DISTINCT c.customer_id) AS total_customers, COUNT(s.transaction_id) AS total_orders, ROUND(SUM(s.revenue), 0) AS total_revenue, ROUND(AVG(s.revenue), 0) AS avg_order_value, ROUND( COUNT(s.transaction_id) / COUNT(DISTINCT c.customer_id), 1 ) AS avg_orders_per_customer, ROUND( SUM(s.revenue) / SUM(SUM(s.revenue)) OVER() * 100, 1 ) AS revenue_share_pct FROM `ecobel_sales.customers` c INNER JOIN `ecobel_sales.transactions` s ON c.customer_id = s.customer_id WHERE s.sale_date BETWEEN '2024-01-01' AND '2024-09-30' GROUP BY c.customer_type, c.territory ORDER BY total_revenue DESC;

Query 03
Seasonal Demand Pattern — Q4 Spike Detection
Analyses monthly revenue trends across product categories to confirm the Q4 seasonal demand spike and calculate optimal pre-season inventory build-up requirements.
seasonal_demand_analysis.sql
Microsoft SQL Server
-- Seasonal demand pattern analysis across 18 months of sales history -- Confirms Q4 spike and calculates inventory build-up recommendation SELECT MONTH(sale_date) AS sale_month, DATENAME(MONTH, sale_date) AS month_name, product_category, SUM(units_sold) AS total_units, SUM(revenue) AS total_revenue, ROUND( AVG(SUM(units_sold)) OVER ( PARTITION BY product_category ), 0 ) AS annual_avg_monthly_units, ROUND( (SUM(units_sold) - AVG(SUM(units_sold)) OVER( PARTITION BY product_category) ) / AVG(SUM(units_sold)) OVER( PARTITION BY product_category ) * 100, 1 ) AS deviation_from_avg_pct FROM sales_transactions WHERE YEAR(sale_date) IN (2023, 2024) GROUP BY MONTH(sale_date), DATENAME(MONTH, sale_date), product_category ORDER BY product_category, sale_month;
