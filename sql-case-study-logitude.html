Query 01
Project Budget vs Actual Spend by Site
This query joins the projects table with the budget_transactions table to calculate total allocated budget against actual spend for each site, computing variance percentage to identify over-budget sites.
budget_variance_by_site.sql
MySQL
-- Budget variance analysis across all active construction sites -- Logitude Projects Limited | Operations Analysis 2022–2023 SELECT p.site_name, p.location, p.project_type, SUM(b.allocated_budget) AS total_budget, SUM(b.actual_spend) AS total_spend, ROUND( ((SUM(b.actual_spend) - SUM(b.allocated_budget)) / SUM(b.allocated_budget)) * 100, 2 ) AS variance_pct, CASE WHEN ((SUM(b.actual_spend) - SUM(b.allocated_budget)) / SUM(b.allocated_budget)) * 100 > 10 THEN 'OVER BUDGET' WHEN ((SUM(b.actual_spend) - SUM(b.allocated_budget)) / SUM(b.allocated_budget)) * 100 < -5 THEN 'UNDER BUDGET' ELSE 'ON TRACK' END AS budget_status FROM projects p INNER JOIN budget_transactions b ON p.project_id = b.project_id WHERE p.status = 'ACTIVE' AND b.transaction_date BETWEEN '2022-04-01' AND '2023-12-31' GROUP BY p.site_name, p.location, p.project_type ORDER BY variance_pct DESC;

Query 02
Duplicate Invoice Detection — Procurement Inconsistency Audit
This query identifies duplicate supplier invoices across all sites by flagging invoice numbers appearing more than once for the same supplier and amount — the most common inconsistency type found (9 cases).
duplicate_invoice_detection.sql
Microsoft SQL Server
-- Detect duplicate supplier invoices across all construction sites -- Used to flag procurement inconsistencies for finance audit WITH DuplicateInvoices AS ( SELECT invoice_number, supplier_name, site_name, invoice_amount, invoice_date, COUNT(*) OVER ( PARTITION BY invoice_number, supplier_name, invoice_amount ) AS occurrence_count, ROW_NUMBER() OVER ( PARTITION BY invoice_number, supplier_name, invoice_amount ORDER BY invoice_date ) AS row_num FROM procurement_invoices WHERE invoice_date BETWEEN '2022-04-01' AND '2023-12-31' ) SELECT invoice_number, supplier_name, site_name, invoice_amount, invoice_date, occurrence_count, CASE WHEN row_num = 1 THEN 'ORIGINAL' ELSE 'DUPLICATE — FLAG FOR AUDIT' END AS invoice_status FROM DuplicateInvoices WHERE occurrence_count > 1 ORDER BY supplier_name, invoice_date;

Query 03
Monthly Project Timeline Performance — On-Time Delivery Tracking
Tracks milestone completion dates against planned dates across all sites to calculate on-time delivery rate and identify which departments were causing the most delays.
timeline_performance.sql
BigQuery
-- Monthly milestone completion rate and delay analysis by department -- Identifies bottlenecks causing scheduling delays across sites SELECT FORMAT_DATE('%Y-%m', planned_completion_date) AS month, responsible_department, COUNT(*) AS total_milestones, COUNTIF(actual_completion_date <= planned_completion_date) AS completed_on_time, COUNTIF(actual_completion_date > planned_completion_date) AS completed_late, ROUND( COUNTIF(actual_completion_date <= planned_completion_date) / COUNT(*) * 100, 1 ) AS on_time_rate_pct, ROUND( AVG(DATE_DIFF(actual_completion_date, planned_completion_date, DAY)), 1 ) AS avg_delay_days FROM `logitude_ops.project_milestones` WHERE planned_completion_date BETWEEN '2022-04-01' AND '2023-12-31' AND actual_completion_date IS NOT NULL GROUP BY month, responsible_department HAVING avg_delay_days > 0 ORDER BY avg_delay_days DESC LIMIT 10;
